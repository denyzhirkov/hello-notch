Ты — senior macOS engineer (Swift + SwiftUI + AppKit), делаешь нативное “widget-like” приложение для macOS (menu bar / background app), которое показывает оверлей-панель под notch. Визуальная цель: создаётся ощущение, что сама чёлка плавно вытягивается вниз, меняется только высота. Пользователь не должен видеть отличий по цвету и ширине — панель должна выглядеть как продолжение чёрной области чёлки.

Главная цель (Phase 1 / MVP)

Сделать минимально рабочее приложение:
	1.	Запускается как background/menu bar app (без Dock, без main window).
	2.	Имеет оверлей-панель, позиционируемую строго по центру верхнего края активного экрана.
	3.	Панель по умолчанию скрыта.
	4.	При вызове showPulse() (событие “пришла задача/инфа”) панель анимированно выезжает вниз, увеличивая высоту от:
	•	collapsedHeight (пример: 18–24px)
	•	до expandedHeight (пример: 36–48px)
(то есть “от половины чёлки до полной чёлки” — реализовать как конфиг).
	5.	Через autoHide (например 4–6 сек) панель уезжает обратно.
	6.	Внутри панели пока достаточно текста-заглушки (например “New item”), но дизайн должен поддерживать будущую фазу (список, действия).

Нефункциональные требования
	•	Максимальная визуальная “интеграция” с notch:
	•	фон панели #000000 (чистый чёрный),
	•	ширина совпадает с notch-area (или визуально эквивалентна): используем центрирование и фиксированную ширину-переменную notchWidthGuess, которая легко настраивается.
	•	скругления снизу: радиус как у notch (пример 10–14px).
	•	без рамки, без тайтлбара, без видимых границ.
	•	Окно должно быть поверх обычных окон, но не ломать систему:
	•	NSPanel / NSWindow без активации приложения (canBecomeKey=false, canBecomeMain=false),
	•	не должно перехватывать клики, когда скрыто (ignoresMouseEvents=true в скрытом состоянии).
	•	Работает на нескольких мониторах:
	•	панель показывается на active screen (где курсор мыши или где key window/menubar).
	•	Работает в fullscreen приложениях:
	•	collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary].

Архитектура

Сделай проект структурно готовым под расширение:
	•	App (SwiftUI) -> AppDelegate (NSApplicationDelegate) для управления окном.
	•	OverlayController:
	•	создаёт OverlayPanel,
	•	имеет методы showPulse(message:), show(expanded:), hide(),
	•	хранит состояние и таймер автоскрытия,
	•	умеет перепозиционировать окно при смене активного экрана/изменении размеров.
	•	OverlayPanel (NSPanel subclass):
	•	настраивает уровень окна, поведение spaces/fullscreen, прозрачность, shadow off, background clear.
	•	NotchView (SwiftUI View):
	•	чёрный фон, нижние скругления, контент заглушка,
	•	плавная анимация высоты (но финальное движение делаем изменением frame окна или constraint).
	•	ScreenResolver:
	•	функция activeScreen() (по позициям mouse location),
	•	helper для вычисления top-center anchor.

Реализация позиционирования
	•	Координаты считать в NSScreen.frame (global).
	•	Верхний край = screen.frame.maxY.
	•	Окно располагается так, чтобы его верхняя граница была “прижата” к top edge:
	•	windowFrame.origin.y = screen.frame.maxY - currentHeight
	•	windowFrame.origin.x = screen.frame.midX - width/2
	•	При анимации меняем высоту окна и origin.y синхронно, чтобы верх “держался” у края, а низ ехал вниз.

Анимации
	•	Используй NSAnimationContext или SwiftUI withAnimation + синхронный апдейт окна.
	•	Два состояния:
	•	collapsed: малая высота
	•	expanded: большая высота
	•	“Пульс” = expanded на N секунд -> collapsed/hidden
	•	Предпочтительно: collapsed держать “почти незаметным” (или полностью hidden). Для Phase 1 можно сделать hidden->expanded->hidden.
